# Glosario de Desarrollo Web

Este documento recopila las definiciones clave de los conceptos tratados en la guía, organizados por módulos de aprendizaje.

---

## Introducción

### Historia
**Historia de la Web**
Evolución de internet desde la Web 1.0 (estática) hasta la Web 3.0. Incluye hitos como la creación de ARPANET, el nacimiento del protocolo HTTP/HTML por Tim Berners-Lee y la explosión de las "puntocom".

### Qué es desarrollo web
**Desarrollo Web**
El proceso integral de construcción y mantenimiento de sitios y aplicaciones web. Abarca desde la codificación simple de texto estático hasta aplicaciones web complejas, comercio electrónico y servicios de redes sociales.

### Tecnologías actuales
**Stack Tecnológico**
Conjunto de lenguajes, frameworks y herramientas que se utilizan actualmente para construir un proyecto (ej. MERN Stack: MongoDB, Express, React, Node). Incluye tendencias modernas como Serverless, PWA (Progressive Web Apps) y WebAssembly.

---

## Frontend (Cliente)

### HTML Básico
**HTML (HyperText Markup Language)**
El lenguaje de marcado estándar utilizado para crear la estructura y el contenido de las páginas web. Define elementos como encabezados, párrafos, enlaces e imágenes.

### CSS Avanzado
**CSS (Cascading Style Sheets)**
Lenguaje de diseño gráfico para definir la presentación de un documento estructurado en HTML. "Avanzado" refiere al uso de maquetación moderna (Flexbox, Grid), animaciones, preprocesadores (Sass/Less) y diseño responsivo (adaptable a móviles).

### JavaScript Moderno
**JavaScript (ES6+)**
Lenguaje de programación interpretado que permite crear contenido dinámico e interactividad en las páginas web. La versión "moderna" incluye características como funciones flecha, promesas, `async/await` y desestructuración.

### Frameworks
**Frameworks de Frontend**
Bibliotecas de código preescrito que proporcionan una estructura base para el desarrollo rápido de interfaces. Los más populares actualmente son React, Vue.js y Angular, los cuales facilitan la creación de SPAs (Single Page Applications).

---

## Backend (Servidor)

### APIs REST
**API REST (Representational State Transfer)**
Un estilo de arquitectura de software que define un conjunto de restricciones para crear servicios web. Permite la comunicación entre el cliente y el servidor, generalmente usando formato JSON y métodos HTTP (GET, POST, PUT, DELETE).

### Bases de datos
**Base de Datos (DB)**
Sistema organizado para recopilar, almacenar y gestionar datos.
* **Relacionales (SQL):** Como MySQL o PostgreSQL, usan tablas estructuradas.
* **No Relacionales (NoSQL):** Como MongoDB, usan formatos flexibles como documentos.

### Seguridad
**Ciberseguridad Web**
Prácticas y herramientas para proteger sitios web y servidores de ataques maliciosos. Incluye conceptos como HTTPS (certificados SSL), encriptación de contraseñas (hashing), protección contra inyecciones SQL y gestión de sesiones (JWT/Cookies).

### Servidores
**Servidor Web**
Software (o hardware) que almacena los archivos de un sitio web (HTML, CSS, imágenes) y los entrega al usuario final cuando este los solicita a través de un navegador. Ejemplos comunes: Apache, Nginx, o entornos de ejecución como Node.js.
